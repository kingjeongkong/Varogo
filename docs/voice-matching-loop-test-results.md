# Voice-Matching Feedback Loop 테스트 결과

테스트 일자: 2026-05-08
테스트 환경: 로컬 개발 서버 (localhost:3000)

---

## 개요

`POST /post-drafts` 요청 시 실행되는 generate → evaluate → regenerate 루프의
실제 동작을 3회 테스트한 결과를 정리한다.

## 테스트 2

**조건:** Today's context 있음 (테스트 1과 다른 내용)

### 1차 evaluate — 3/3 실패

3개 옵션 공통 실패 원인:

- 콜론(`:`) 사용 — 레퍼런스 포스트에 콜론 없음
- 문장 구조 과복잡
- 너무 formal한 톤

### 2차 re-evaluate (retry 후) — 3/3 전부 재실패 (진자 운동)

| 옵션    | 결과 | 실패 이유                                                                |
| ------- | ---- | ------------------------------------------------------------------------ |
| option1 | 실패 | 느낌표 사용, 문장구조 과단순 (과교정), 시그니처 구문 "In the end" 미사용 |
| option2 | 실패 | 느낌표 사용, 추상적 개념 과다, 시그니처 구문 "In the end" 미사용         |
| option3 | 실패 | 느낌표 사용, 문장구조 과단순 (과교정), 시그니처 구문 "In the end" 미사용 |

**주목할 점:** 1차에서 "문장이 너무 복잡하다"는 지적을 받아 retry가 단순화를 시도했으나,
반대 방향으로 과교정되어 "너무 단순하다"로 재실패. 전형적인 진자 운동 패턴.
또한 retry 중 느낌표 금지 규칙을 새로 위반 — retry 프롬프트가 formatting habits를 충분히 강제하지 못한 결과.

## 패턴 및 시사점

### 반복되는 1차 실패 원인

- **느낌표 사용** — 4번 모두 등장. 레퍼런스에 없어도 LLM이 자주 삽입하는 가장 흔한 위반
- **오프닝 패턴 불일치** — 테스트 1~3에서 반복. GPT가 가장 못 지키는 구조적 규칙
- **톤 불일치** — 과도한 열정 or conversational 톤이 레퍼런스와 다름

### Retry의 한계

- **과교정 (테스트 2, 4)** — 특정 방향으로 고치다가 반대 방향으로 새 문제 생성
- **Formatting habits 미강제** — retry 프롬프트가 느낌표/콜론 같은 포맷 규칙을 충분히 재강조하지 않아 동일 위반 반복
- **Hallucination (테스트 1 option3)** — evaluator가 잡아내긴 했으나 retry 후에도 완전 수정 안 됨

### Today's context의 영향

- context 내용에 따라 LLM 출력 스타일이 달라지고, retry 성공률도 변동
- 테스트 3에서 retry 성공률이 가장 높았던 것은 context가 LLM이 voice를 맞추기 더 쉬운 방향으로 작성됐을 가능성이 있음

### 개선 방향 (프롬프트 레벨)

- retry 프롬프트에서 formatting habits (느낌표, 콜론 등) 를 명시적으로 재강조
- 과교정 방지를 위해 retry 프롬프트에 "원본의 문장 복잡도를 유지하되 X만 고쳐라" 방향으로 구체화
